🤖 Transformer Tokenizer Explained

See how different tokenizers break down text into tokens

1 Text Preprocessing

The raw text is normalized (lowercasing, unicode normalization, etc.) and prepared for tokenization.

2 Tokenization

Text is split into tokens using the selected algorithm. Each tokenizer has different rules for splitting.

3 Token → ID Conversion

Each token is mapped to a unique integer ID from the model's vocabulary.

4 Embedding Lookup

Each token ID is converted to a dense vector representation (embedding) that the model can process.

📊 Tokenizer Comparison
Algorithm Used By Strategy Key Feature
BPE GPT, GPT-2, GPT-3, RoBERTa Merge most frequent byte pairs Handles rare words well
WordPiece BERT, DistilBERT Maximize likelihood of training data Uses ## prefix for subwords
Unigram T5, ALBERT, XLNet Probabilistic segmentation Multiple possible tokenizations
Character Some small models Split into individual characters No unknown tokens, very long sequences
